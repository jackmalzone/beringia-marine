# Insights Blog SEO Implementation

This document describes the SEO metadata and structured data implementation for the Insights blog system.

## Overview

The Insights blog system includes comprehensive SEO optimization with:

- Custom metadata for listing and article pages
- JSON-LD structured data (Article, Blog, BreadcrumbList schemas)
- Open Graph tags for social media sharing
- Twitter Card meta tags
- Proper canonical URLs
- Automatic fallbacks for missing SEO fields

## Metadata Configuration

### Insights Listing Page

**Location:** `src/app/insights/page.tsx`

The insights listing page includes:

- **Title:** "Wellness Insights & Recovery Research | Vital Ice"
- **Description:** Comprehensive description of the blog content
- **Keywords:** wellness articles, recovery research, cold therapy benefits, etc.
- **Open Graph:** Custom OG tags with dedicated image
- **Twitter Card:** Summary large image card
- **Structured Data:**
  - Blog schema
  - BreadcrumbList schema

### Individual Article Pages

**Location:** `src/app/insights/[slug]/page.tsx`

Each article page includes:

- **Dynamic Title:** Article title + "| Vital Ice Insights" (or custom SEO title)
- **Dynamic Description:** Article abstract (or custom SEO description)
- **Keywords:** Article tags
- **Open Graph:**
  - Type: "article"
  - Published time
  - Authors
  - Tags
  - Custom or cover image
- **Twitter Card:** Summary large image with article details
- **Structured Data:**
  - Article schema with full metadata
  - BreadcrumbList schema (Home > Insights > Article)

## Custom SEO Fields

Articles support custom SEO fields through the `seo` property:

```typescript
interface ArticleSEO {
  title?: string; // Custom page title
  description?: string; // Custom meta description
  ogImage?: string; // Custom Open Graph image
  keywords?: string[]; // Custom keywords (optional)
}
```

### Fallback Behavior

If custom SEO fields are not provided, the system automatically falls back to:

- **Title:** `${article.title} | Vital Ice Insights`
- **Description:** `article.abstract`
- **OG Image:** `article.coverImage`
- **Keywords:** `article.tags`

## Implementation Details

### Metadata Generation

The `generateMetadata()` function in article pages:

1. Fetches the article by slug
2. Returns 404 metadata if article not found
3. Merges base metadata with article-specific metadata
4. Includes Open Graph and Twitter Card tags
5. Uses custom SEO fields with fallbacks

### Structured Data Injection

JSON-LD scripts are injected in the page component:

```tsx
<script
  type="application/ld+json"
  dangerouslySetInnerHTML={{
    __html: generateStructuredData(schema),
  }}
/>
```

This approach:

- Renders on the server (SSR/SSG)
- Is crawlable by search engines
- Validates with Google's Rich Results Test
- Supports multiple schemas per page

## SEO Best Practices

### 1. Unique Titles and Descriptions

Each article has a unique title and description, either custom or auto-generated from article content.

### 2. Semantic HTML

Articles use proper heading hierarchy (h1 for title, h2-h4 for content).

### 3. Image Optimization

- All images have descriptive alt text
- Cover images are optimized for social sharing (1200x630px)
- Images are served from CDN with lazy loading

### 4. Canonical URLs

Canonical URLs are automatically set through Next.js metadata configuration.

### 5. Mobile-Friendly

All pages are responsive and mobile-optimized.

### 6. Performance

- ISR (Incremental Static Regeneration) with 1-hour revalidation
- Static generation for all article pages
- Optimized images and code splitting
